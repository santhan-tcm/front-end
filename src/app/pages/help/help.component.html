 Frequently Asked Questions

Which message versions are supported?

The validator is version-blind. It uses the message family (e.g., pacs.008) to find the most compatible structural blueprint, ensuring maximum support for all SR releases.

Is my data secure?

All validations are processed in a high-security environment. If 'Store History' is toggled off, no record of your message will be saved in our audit trail.

What does Layer 5 validation check?

Layer 5 checks dynamic reference data, such as verifying BIC codes against the global directory and validating currency codes against ISO 4217 standards.
